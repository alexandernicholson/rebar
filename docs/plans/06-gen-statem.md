# Plan: GenStatem (State Machine)

## Summary

Implement Erlang's `gen_statem` — a generic state machine behavior with state-enter callbacks, per-state timeouts, event postponement, and typed state transitions. More structured than GenServer for protocol implementations, connection managers, and workflow engines.

## Rebar Integration Points

- **Runtime**: Uses `Runtime::spawn()` for the state machine process
- **ProcessId**: State machines are first-class processes with PIDs
- **MessageRouter**: State machines send/receive via standard routing
- **GenServer patterns**: Similar engine structure but different callback model
- **Timer module**: State timeouts and event timeouts use timer infrastructure

## Proposed API

```rust
/// Callback mode for the state machine.
#[derive(Debug, Clone, Copy, PartialEq, Eq)]
pub enum CallbackMode {
    /// Each state is a variant of the State enum.
    /// Dispatches to handle_event with the current state.
    HandleEventFunction,
}

/// Event types that the state machine handles.
#[derive(Debug, Clone)]
pub enum EventType<Call> {
    /// Synchronous call (expects a reply).
    Call(oneshot::Sender<Call>),
    /// Asynchronous cast (fire-and-forget).
    Cast,
    /// Raw info message from mailbox.
    Info,
    /// State timeout expired.
    StateTimeout,
    /// Generic named timeout expired.
    Timeout(String),
    /// Event timeout expired.
    EventTimeout,
    /// Internal event (generated by next_event action).
    Internal,
    /// State enter event.
    Enter { old_state_name: String },
}

/// Actions the state machine can take during a transition.
#[derive(Debug)]
pub enum Action<State, Reply> {
    /// Send a reply to a caller.
    Reply(oneshot::Sender<Reply>, Reply),
    /// Set a state timeout (cancelled on state change).
    StateTimeout(Duration, rmpv::Value),
    /// Set an event timeout (cancelled on any event).
    EventTimeout(Duration, rmpv::Value),
    /// Set a named generic timeout.
    GenericTimeout(String, Duration, rmpv::Value),
    /// Cancel a timeout.
    CancelTimeout(TimeoutKind),
    /// Postpone the current event (re-queue on state change).
    Postpone,
    /// Insert an internal event into the queue.
    NextEvent(rmpv::Value),
    /// Hibernate the process (hint for memory optimization).
    Hibernate,
}

#[derive(Debug, Clone)]
pub enum TimeoutKind {
    State,
    Event,
    Generic(String),
}

/// Result of handling an event.
pub enum TransitionResult<State, Data, Reply> {
    /// Transition to a new state with new data.
    NextState {
        state: State,
        data: Data,
        actions: Vec<Action<State, Reply>>,
    },
    /// Keep the current state, update data.
    KeepState {
        data: Data,
        actions: Vec<Action<State, Reply>>,
    },
    /// Keep state and data unchanged, optionally with actions.
    KeepStateAndData {
        actions: Vec<Action<State, Reply>>,
    },
    /// Stop the state machine.
    Stop {
        reason: ExitReason,
        data: Data,
    },
    /// Stop with pending replies.
    StopAndReply {
        reason: ExitReason,
        data: Data,
        replies: Vec<(oneshot::Sender<Reply>, Reply)>,
    },
}

/// The GenStatem trait. Implement this for your state machine.
#[async_trait]
pub trait GenStatem: Send + Sync + 'static {
    /// The state type. Each variant represents a distinct state.
    type State: Send + Clone + PartialEq + fmt::Debug + 'static;
    /// Mutable data carried across states.
    type Data: Send + 'static;
    /// Synchronous call message type.
    type Call: Send + 'static;
    /// Asynchronous cast message type.
    type Cast: Send + 'static;
    /// Reply type for calls.
    type Reply: Send + 'static;

    /// Return the callback mode. If state_enter is true,
    /// handle_event will be called with EventType::Enter on state changes.
    fn callback_mode(&self) -> (CallbackMode, bool); // (mode, state_enter)

    /// Initialize the state machine. Returns initial (state, data).
    async fn init(&self) -> Result<(Self::State, Self::Data), String>;

    /// Handle all events. Dispatched based on current state and event type.
    async fn handle_event(
        &self,
        event_type: EventType<Self::Reply>,
        event: rmpv::Value,
        state: &Self::State,
        data: &mut Self::Data,
    ) -> TransitionResult<Self::State, Self::Data, Self::Reply>;

    /// Called when the state machine is shutting down.
    async fn terminate(
        &self,
        _reason: ExitReason,
        _state: &Self::State,
        _data: &mut Self::Data,
    ) {}
}

/// A handle to a running state machine.
#[derive(Clone)]
pub struct GenStatemRef<S: GenStatem> {
    pid: ProcessId,
    call_tx: mpsc::Sender<StatemCallEnvelope<S>>,
    cast_tx: mpsc::UnboundedSender<S::Cast>,
}

impl<S: GenStatem> GenStatemRef<S> {
    pub fn pid(&self) -> ProcessId;

    /// Synchronous call to the state machine.
    pub async fn call(
        &self,
        msg: S::Call,
        timeout: Duration,
    ) -> Result<S::Reply, CallError>;

    /// Asynchronous cast to the state machine.
    pub fn cast(&self, msg: S::Cast) -> Result<(), SendError>;
}

/// Spawn a state machine process.
pub async fn spawn_gen_statem<S: GenStatem>(
    runtime: Arc<Runtime>,
    statem: S,
) -> GenStatemRef<S>;
```

## Elixir/Erlang Reference Documentation

### Erlang gen_statem Module (stdlib)

#### Overview

The `gen_statem` behavior provides a generic state machine framework replacing `gen_fsm` since OTP 20.0. It supports two callback modes and offers advanced features like event postponing, state timeouts, and automatic state enter calls.

#### Callback Modes

**state_functions**: State must be an atom; each state has its own callback function.

**handle_event_function**: State can be any term; all events route through a single `handle_event/4`.

Both modes optionally support `state_enter` calls triggered on state changes.

#### Required Callbacks

- **`init(Args)`**: Returns `{ok, State, Data}` or error variants
- **`callback_mode()`**: Returns `state_functions | handle_event_function`, optionally with `state_enter`
- **`terminate(Reason, CurrentState, Data)`**: Cleanup on shutdown

#### Event Types

**External events**: `{call, From}`, `cast`, `info`
**Timeout events**: `timeout`, `{timeout, Name}`, `state_timeout`
**Internal events**: Generated via `next_event` actions

#### Transition Actions (Return Values)

- `{next_state, NextState, NewData [, Actions]}`
- `{keep_state, NewData [, Actions]}`
- `keep_state_and_data | {keep_state_and_data, Actions}`
- `{repeat_state, NewData [, Actions]}` (with state_enter)
- `{stop, Reason [, NewData]}`
- `{stop_and_reply, Reason, Replies [, NewData]}`

#### Available Actions

**Postponement**: `postpone | {postpone, boolean()}`
**Event insertion**: `{next_event, EventType, EventContent}`
**Hibernation**: `hibernate | {hibernate, boolean()}`

**Timeouts**:
- Event timeout: `{timeout, Time, EventContent}` — cancelled by any event
- Generic timeout: `{{timeout, Name}, Time, EventContent}` — per-name, independent
- State timeout: `{state_timeout, Time, EventContent}` — cancelled on state change
- Cancel: `{timeout, cancel}`, `{state_timeout, cancel}`, etc.
- Update: `{timeout, update, NewContent}`

**Replies**: `{reply, From, Reply}`

#### State Enter Calls

When `callback_mode()` includes `state_enter`, the state callback is invoked as `(enter, OldState, Data)` on state changes. Enter calls cannot postpone events or change states.

#### Key Concepts

**State transitions**: `NextState =/= CurrentState` triggers postponed event retries and state enter calls.
**Event queue**: Manages external events, postponed events, and inserted events with deterministic ordering.
**Timeout semantics**: Event timeouts cancel on any arrival; state timeouts cancel on state change; generic timeouts per-name with restart capability.
